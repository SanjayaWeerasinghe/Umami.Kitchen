# Umami.Kitchen

A single-page marketing site for a fictional healthy-bowl subscription. Built as a demo of a scroll-scrubbed video hero: the salad assembles itself as you scroll, five story beats fade in and out over the top, and the rest of the page reads normally on a warm cream palette.

Rebuilt on **Astro** with a mobile fallback so phones no longer stall on the 5 MB video.

## Stack

- **[Astro 5](https://astro.build/)** — HTML-first, zero JS by default
- **[Tailwind CSS](https://tailwindcss.com/)** — via `@astrojs/tailwind`
- **[GSAP + ScrollTrigger](https://gsap.com/)** — scroll-driven progress
- **[ffmpeg-static](https://www.npmjs.com/package/ffmpeg-static)** — bundled ffmpeg for poster + video optimization (dev only)
- Fraunces + Inter from Google Fonts

## Setup

```bash
npm install
npm run poster   # extract public/bowl-poster.jpg + optimize bowl.mp4 with +faststart
npm run dev      # http://localhost:4321
```

## Build & deploy

```bash
npm run build    # → dist/
npm run preview  # serve dist/ locally
```

Deploy the `dist/` folder anywhere: Cloudflare Pages, Vercel, Netlify, GitHub Pages, or any static host. All three of the first serve `Range` requests correctly, which is what mobile browsers need to stream (and scrub) the video.

## How the scroll-scrubbed video works

1. `<video>` is `position: fixed`, `object-fit: cover`, full viewport, `z-index: 0`.
2. Once `loadeddata` fires, a single `play() → pause()` cycle warms the decoder (Chromium on Windows shows a black frame otherwise).
3. A ScrollTrigger with `scrub: true` on a track sized to roughly match `video.duration` (`duration × 60vh`, clamped) maps scroll progress to `video.currentTime` on every frame.
4. Five story beats live over the video inside the pinned section, toggled by explicit progress thresholds.
5. Sections below the story (ingredients, nutrition, FAQ, CTA) have solid backgrounds and `z-10`, so they visually cover the fixed video as they scroll into view.

## Mobile fallback

Video scrubbing is expensive on mobile and outright fails on some devices (Safari refuses `currentTime` seeks on paused videos in Low Power Mode; Chrome on Android skips `preload="auto"` on cellular). `BowlStage.astro` detects any of the following and swaps the video for the poster JPG:

- **Viewport ≤ 768px** — phone / narrow tablet
- **`prefers-reduced-motion: reduce`**
- **`prefers-reduced-data: reduce`**
- **`navigator.connection.saveData === true`** or effectiveType `2g` / `slow-2g`
- **Video `error` event** — network or codec failure

In fallback mode, the story beats still fade in as you scroll — the pinned section just uses a fixed height (400vh) and the video timeline is ignored.

## Video optimization notes

The `npm run poster` script does two things:
1. Extracts `public/bowl-poster.jpg` from ~0.5s into the video (skips any black lead-in)
2. Re-encodes `public/bowl.mp4` with:
   - `-movflags +faststart` → moov atom at the front so browsers can start playing before the file finishes downloading
   - `-crf 23 -preset medium` → decent quality/size trade-off
   - No audio (`-an`) — this is a background video

If scrubbing still feels choppy under continuous wheel input on desktop, re-encode with a keyframe on every frame (larger file but instant seeks):

```bash
node -e "const p = require('ffmpeg-static'); require('child_process').execFileSync(p, ['-y','-i','public/bowl.mp4','-c:v','libx264','-crf','20','-g','1','-preset','slow','-movflags','+faststart','-an','public/bowl-scrubbable.mp4'], {stdio:'inherit'});"
```
